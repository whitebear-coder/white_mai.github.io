计算机视觉面试高频考点

卷积神经网络基础知识

  • 卷积运算， 计算卷积参数量， 1 * 1卷积核 作用
  • 感受野：卷积神经网络每一层输出的特征图上的像素点在输入图片上映射区域的大小.通俗的解释：特征图上的一个点对应输入图像上的区域
  • 反卷积
  • 空洞卷积（实时图像分割）补零，增大感受野。
  • 卷积的时间复杂度等
    • 时间复杂度：M：输出特征图， K：卷积核 Cin:输入通道数 Cout:输出通道数 Time ~O(M^2 K^2 Cin * Cout)

经典卷积神经网络(AlexNet, Inception, VGG, ResNet)

  • Inception v1 将1 1， 3 3， 5 5的conv和3 3 pooling堆叠一起，一方面增加了网络的width, 另一方面增加了网络对尺度的适应性，增加了网络的宽度，增加了网络对尺度的适应性，不同支路的感受野是不同的，所以有多尺度的信息在里面。

  • Inception v2 加入了BN层，减少了Internal Covariate Shift, 使每一层输出都规范化到一个N(0, 1)的高斯， 用两个3 3conv 替代了inception模块中的55， 即降低了参数数量，也加速计算。

  • Inception v3 分解 将7 7分解成两个一维的卷积(1 7, 7 1), 3 3也是一样（1 3， 3 1) 这样的好处，既可以加速计算，也可以将1个conv 拆成 2个conv 使得网络深度进一步增加，增加了网络的非线性

  • Inception v4 ResNet

  • VGG：采用连续的几个3 * 3的卷积核代替AlexNet中的较大卷积核： 多层非线性可以增加网络深度来保证学习更复杂的模式，而且代价比较小(参数更少)

    • 两个3 3代替1个5 5， 三个3 3 代替1个7 7(卷积基本运算可得)

    • 优点：

      • VGGNet的结构非常简洁，整个网络都使用了同样大小的卷积核尺寸（3x3）和最大池化尺寸（2x2）。

      • 几个小滤波器（3x3）卷积层的组合比一个大滤波器（5x5或7x7）卷积层好

      • 验证了通过不断加深网络结构可以提升性能。

    • 缺点：

      • 内存占的多（140M）绝大多数参数来自于第一个全连接层，VGG一共有三个全连接层

  • ResNet

    • 减少信息冗余性：relu适合高维不适合低维
    • 解决问题：解决网络深度增加到一定程度，更深的网络堆叠效果反而变差的问题，在网络深度到一定程度，误差升高，效果变差，梯度消失现象明显，训练变差，ResNet解决深层网络梯度消失问题
    • 优点：使前馈/反馈传播算法顺利进行，结构更加简单，恒等映射增加基本不会降低网络的性能
    • 缺点：训练时间长。

轻量级神经网络（ShuffleNet, MobileNet)

ShuffleNet

  • 设计理念: 对不同的channels进行shuffle来解决group convolution带来的弊端，Group convolution 是将输入层不同特征图进行分组，然后采用不同卷积和对各组进行卷积，降低卷积的计算量。

    • 通道密集连接方式：一般的卷积都是在所有的输入特征图上做卷积，全通道卷积。
    • 通道稀疏连接方式：group convolution
    • group convolution存在弊端：
      • 将channel分组，不同组之间的feature map是不通信的，目测会降低网络的特征提取能力。利用1*1卷积降维就可以达到不同channel之间的通信能力。

  • ShuffleNet 特点：对group convolution之后的特征图进行重组 shuffle过程并不是随机的，(n, g) ->(g, n) ->reshape成一个维度

  • ShuffleNet 网络结构：

    • Depthwise Convolution:

      • 与传统卷积区别

        • 一个三通道卷积核与一个RGB图像卷积，生成一个新的三维特征图

        • DW使用三个单通道的卷积进行运算，得到三张同样大小的特征图(简单来说就是DW卷积的kernel为单层，因此输出的通道数等于输入图片的通道数)因此缺少通道间的信息交流，所以需要下面的 pointwise convolution

    • Pointwise Convolution:

      • PW卷积核的尺寸为1 1 B B为上一层的通道数，就是在通道上对输入的特征进行加权整合

    • 优势

      • 参数少，计算速度快，参数对比如下

        • 传统卷积参数个数为：4 3 3 * 3 = 108

      • depthwise convolution 和 pointwise convolution

    • ShuffleNet 网络结构：

      • 一层11 Gconv ->channel shuffle ->一层 3 3 DWConv ->一层 1 * 1GConv

MobileNet

  • MobileNet v1: MobileNet v1就是 把VGG中的标准卷积层换成深度可分离卷积

    • 空间可分离：将一个大的卷积核变成两个小的卷积核，比如将3 3的卷积核矩阵转成两个1 3的卷积向量
    • 深度可分离卷积 : 深度可分离卷积 ＝ 深度卷积 + 逐点卷积

  • MobileNet v2:

    • 网络的亮点：1.Inverted Residuals(倒残差结构) 2.Linear Bottlenecks
    • 倒残差结构：
      • 残差结构：1 1卷积降维 -> 3 3卷积 -> 1 * 1卷积升维
      • 倒残差结构： 1 1卷积升维-> 3 3卷积DW -> 1 * 1卷积降维 激活函数使用relu6
        • 不使用relu激活函数的原因：relu激活函数对低维的feature map造成的损失较大

  • MobileNet v3:

    • 改进：

      • 更新的block, 在倒残差结构上改动 -> 加入SE模块，更新了激活函数

        • SE Block的机制：

          如上图，有黄蓝两个特征矩阵，先对其进行平均池化得到0.25和0.3。然后经过两个全连接层，第一个全连接层采用Relu激活函数，第二个全连接层采用Hard-sigmoid激活函数，然后将两个值变为0.5和0.6。将得到的两个值与原矩阵的每个元素相乘，得到新的特征矩阵（如黄色矩阵第一格0.2与0.5相乘，得到0.1；蓝色矩阵第一格0.2*0.6=0.12，以此类推）。

      • 使用NAS搜索参数(神经网络结构设置)

      • 重新设计耗时层结构

        • 减少第一个卷积层卷积核的个数
        • 精简Last Stage

图像分割任务逻辑以及实用模型

  • UNet
    • 多尺度获得信息
    • 神来之笔-skip connection
  • FCN(全卷积神经网络)
    • 优点
      • 可以 接受任意大小的输入图像，而不用要求所有的训练图像和测试图像具有同样的尺寸。
      • 更加高效，因为避免了由于使用像素块而带来的重复存储和计算卷积的问题。
    • 缺点
      • 得到的结果还是不够精细。进行8倍上采样虽然比32倍的效果好了很多，但是上采样的结果还是比较模糊和平滑， 对图像中的细节不敏感。
      • 对各个像素进行分类，没有充分考虑像素与像素之间的关系，忽略了在通常的基于像素分类的分割方法中使用的空间规整（spatial regularization）步骤，缺乏空间一致性。
  • Mask R-CNN 其实就相当于Faster RCNN + Mask（FCN），也就是在Faster RCNN的基础上，增加了一个分支用作图像分割。

数据增强

  • 作用：
    • 避免过拟合
    • 提升模型鲁棒性
    • 增加训练数据
    • 避免样本不均衡
  • 常用方法
    • 几何变换方法：翻转，旋转，裁剪，缩放，平移，抖动。
    • 像素变换方法：加椒盐噪声，高斯噪声，高斯模糊，调整HSV对比度，调节亮度，饱和度，直方图均衡化，调整白平衡等。
  • 适合分类任务的几个数据增强方式
    • mixup：将随机的两张样本按比例混合，分类的结果按比例分配。只适合分类任务。
    • cutout：随机的将样本中的部分区域cut掉，并且填充0像素值，分类的结果不变。
    • cutmix：就是将一部分区域cut掉但不填充0像素而是随机填充训练集中的其他数据的区域像素值，分类结果按一定的比例分配。
  • 适合检测任务的几个增强方式
    • gridmask 网格掩码区域

batch normalization ->见base DL博客

  • 补充问题：
    • BN在训练和测试阶段不同
      • 训练时：是对每一个batch的训练数据进行归一化，也就是用每一批数据的均值和方差。
      • 测试时：对单个样本进行测试，不存在batch的概念，这时均值和方差是全部训练数据的均值和方差，这两个数值是通过 移动平均法 求得。
    • BN训练时如果使用全量训练集的均值和方差的话，会容易导致过拟合

反向传播算法

Adam SGD

  • SGD
    • 大部分时候会向着全局最小值靠近，有时候会远离最小值，恰好这个样本指向方向不对(SGD是有很多噪声的)， 平均来看，最终会靠近最小值，不过有时候也会方向错误，因为SGD永远不会收敛，而是会一直在最小值附近波动。一次只处理一个样本，这样效率过于低下。
  • Adam

损失函数优缺点

类别不均衡问题

  • 欠采样和过采样
    • 过采样：少数类别数量重复利用，不过这种方法不推荐，可能会导致过拟合
    • 欠拟合：数据量大的数据不是全部使用，正负样本数据尽量平衡

  • 合并新的少数类别：比如自行车和摩托车改为非机动车
  • 调整类权重

  • Xavier初始化：保持输入和输出的方差一致
